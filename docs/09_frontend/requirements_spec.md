# 요구사항 정의서

## 1. 개요

ZIPSA 프론트엔드(Next.js App Router) 및 백엔드(FastAPI)의 기능 요구사항을 정의합니다.
와이어프레임(`wireframe_spec.md`) 및 API 명세(`zipsa_openapi.yaml`) 기반으로 작성합니다.

### 1.1 사용자 분류

| 구분 | 설명 | 접근 범위 |
|---|---|---|
| **게스트** | 미인증 사용자 | 채팅(비개인화), 품종 탐색, 냥심 번역기 |
| **신규 유저** | 인증 완료, 온보딩 미완료 | 온보딩 전용 |
| **일반 유저** | 인증 완료, 온보딩 완료 | 전체 기능 |

### 1.2 우선순위 기준

| 등급 | 기준 |
|---|---|
| **Must** | 서비스 핵심 기능. 없으면 서비스 불가 |
| **Should** | 중요하지만 초기 릴리즈에서 누락 가능 |
| **Could** | 확장 기능. 여건이 되면 구현 |

## 2. 인증 (Auth)

### FR-AUTH-01 소셜 로그인
- **설명**: Google, Kakao, Naver OAuth를 통해 로그인한다.
- **우선순위**: Must
- **처리 흐름**:
  1. `/auth/signin` 진입 → SSO 버튼 클릭
  2. NextAuth가 OAuth 처리 → `users` 컬렉션 자동 생성
  3. 신규 유저 → `/onboarding` 리다이렉트
  4. 기존 유저 → `/chat` 리다이렉트
- **관련 이슈**: #25

### FR-AUTH-02 인증 상태 유지
- **설명**: JWT 세션을 기반으로 인증 상태를 유지하고, 만료 시 재로그인을 유도한다.
- **우선순위**: Must
- **관련 이슈**: #20

### FR-AUTH-03 게스트 채팅 허용
- **설명**: 미인증 사용자도 채팅을 사용할 수 있다. 단, 세션이 저장되지 않으며 개인화 추천이 적용되지 않는다.
- **우선순위**: Should
- **제한사항**:
  - 채팅 히스토리 저장 없음
  - `user_profile` 컨텍스트 없이 에이전트 동작
  - 채팅창 상단 로그인 유도 넛지 노출

## 3. 온보딩 (Onboarding)

### FR-ONBOARD-01 4단계 온보딩 설문
- **설명**: 신규 유저가 집사 환경 정보를 4단계에 걸쳐 입력한다.
- **우선순위**: Must
- **단계 구성**:

| 단계 | 입력 항목 | 수집 필드 |
|---|---|---|
| 1 나 소개 | 닉네임 | `nickname` |
| 2 우리 집 | 거주 형태, 하루 부재 시간 | `housing`, `work_style` |
| 3 함께 사는 | 동거인, 알레르기 | `companion`, `has_cat`, `allergy` |
| 4 고양이 취향 | 활동량, 경험, 원하는 성향 | `activity`, `experience`, `traits` |

- **관련 이슈**: #27

### FR-ONBOARD-02 동거인 조건부 표시
- **설명**: "혼자 사시나요?" Yes/No 선택 시, No인 경우에만 동거인 선택지(어린 아이 / 강아지 / 가족 / 고양이 있음)를 표시한다.
- **우선순위**: Must

### FR-ONBOARD-03 온보딩 완료 후 프로필 저장
- **설명**: 온보딩 완료 시 `POST /api/v1/users/me/profile`을 호출하여 `user_profiles` 컬렉션에 저장한다.
- **우선순위**: Must
- **관련 이슈**: #28

---

## 4. 채팅 (Chat)

### FR-CHAT-01 새 채팅 세션 생성
- **설명**: "새 채팅" 시작 시 `POST /api/v1/users/me/sessions`를 호출하여 세션을 생성하고, `/chat/[sessionId]`로 이동한다.
- **우선순위**: Must
- **관련 이슈**: #33

### FR-CHAT-02 스트리밍 응답
- **설명**: `POST /api/v1/chat/stream` SSE를 통해 AI 응답을 실시간 스트리밍으로 표시한다.
- **우선순위**: Must

### FR-CHAT-03 메시지 히스토리 로드
- **설명**: 세션 재진입 시 `GET /api/v1/users/me/sessions/{sessionId}/messages`로 이전 메시지를 불러온다.
- **우선순위**: Must
- **관련 이슈**: #34

### FR-CHAT-04 품종 카드 렌더링
- **설명**: Matchmaker 응답 시 우측 패널에 품종 카드(최대 3종)를 렌더링한다. 카드에는 이미지, 품종명, 태그, 설명이 포함된다.
- **우선순위**: Must
- **관련 이슈**: #29, #30

### FR-CHAT-05 구조묘 카드 렌더링
- **설명**: Liaison 응답 시 우측 패널에 구조묘 카드를 렌더링한다. 카드에는 이미지, 보호소명, 성별/나이/중성화 여부, 입양 문의 버튼이 포함된다.
- **우선순위**: Should
- **관련 이슈**: #29, #30

### FR-CHAT-06 RAG 출처 문서 표시
- **설명**: AI 응답 하단에 참고한 출처 문서(제목, 소스, URL)를 표시한다.
- **우선순위**: Should

### FR-CHAT-07 인터랙티브 텍스트
- **설명**: AI 응답 내 고양이 품종명에 밑줄을 표시하고, 호버 시 미니 품종 카드 팝오버를 노출한다.
- **우선순위**: Should
- **관련 이슈**: #31

### FR-CHAT-08 입력창 품종명 자동완성
- **설명**: 채팅 입력창에 품종명 입력 시 `GET /api/v1/breeds?q=`를 호출하여 67종 자동완성 드롭다운을 표시한다.
- **우선순위**: Could
- **관련 이슈**: #31

### FR-CHAT-09 세션 삭제
- **설명**: 전역 Drawer 세션 목록에서 세션을 삭제할 수 있다. `DELETE /api/v1/users/me/sessions/{sessionId}` 호출.
- **우선순위**: Should

## 5. 전역 Drawer

### FR-DRAWER-01 전역 내비게이션
- **설명**: 햄버거 아이콘 클릭 시 전역 Drawer가 슬라이드 인된다. 새 채팅, 채팅 히스토리, 품종 탐색, 냥심 번역기 링크를 포함한다.
- **우선순위**: Must

### FR-DRAWER-02 채팅 히스토리 (인증 유저)
- **설명**: 인증 유저의 경우 Drawer 내 세션 목록을 표시한다. 세션 클릭 시 해당 채팅으로 이동한다.
- **우선순위**: Must

### FR-DRAWER-03 게스트 빈 상태
- **설명**: 미인증 사용자의 경우 세션 목록 대신 "로그인하면 대화 기록이 저장돼요" 안내와 로그인 버튼을 표시한다.
- **우선순위**: Should

---

## 6. 프로필 관리

### FR-PROFILE-01 기본 정보 수정
- **설명**: 닉네임, 나이, 성별, 연락처, 주소를 수정할 수 있다. `PUT /api/v1/users/me/profile` 호출.
- **우선순위**: Should
- **관련 이슈**: #28

### FR-PROFILE-02 주소 입력 (Kakao 우편번호 API)
- **설명**: "주소 검색" 버튼 클릭 시 Kakao 우편번호 팝업을 열어 도로명 주소를 자동 입력한다. 상세주소는 직접 입력한다.
- **우선순위**: Should

### FR-PROFILE-03 양육 환경 수정
- **설명**: 온보딩에서 입력한 6개 항목(거주 형태, 부재 시간, 동거인, 알레르기, 활동량, 성향)을 프로필 페이지에서 수정할 수 있다.
- **우선순위**: Should

---

## 7. 고양이 관리

### FR-CAT-01 고양이 등록
- **설명**: 이름, 나이, 성별, 묘종(67종 자동완성), 사진, 예방접종 기록을 입력하여 고양이를 등록한다. `POST /api/v1/users/me/cats` 호출.
- **우선순위**: Should
- **관련 이슈**: #37

### FR-CAT-02 고양이 수정 및 삭제
- **설명**: 등록된 고양이 정보를 수정하거나 삭제할 수 있다.
- **우선순위**: Should

### FR-CAT-03 예방접종 다중 입력
- **설명**: 고양이 등록/수정 시 예방접종 기록(종류 + 날짜)을 복수로 추가할 수 있다.
- **우선순위**: Could

## 8. 냥심 번역기

### FR-MEME-01 고양이 사진 업로드
- **설명**: 사용자가 고양이 사진을 업로드한다. 드래그 앤 드롭 또는 클릭 선택 지원.
- **우선순위**: Should
- **관련 이슈**: #36

### FR-MEME-02 첨언 입력
- **설명**: 사진 업로드 후 선택적으로 한 마디 첨언(최대 100자)을 입력할 수 있다. 첨언은 Vision LLM 컨텍스트로 주입된다.
- **우선순위**: Should

### FR-MEME-03 밈 텍스트 생성
- **설명**: `POST /meme/generate` 호출 시 Vision LLM이 사진과 첨언을 분석하여 시니컬한 밈 텍스트를 생성한다.
- **우선순위**: Should
- **관련 이슈**: #36

### FR-MEME-04 폴라로이드 렌더링
- **설명**: 생성된 밈 텍스트를 폴라로이드 카드 형태로 렌더링한다.
- **우선순위**: Should
- **관련 이슈**: #35

### FR-MEME-05 1-Click 고양이 등록
- **설명**: 밈 결과물에서 "내 고양이로 등록" 버튼 클릭 시 해당 사진으로 고양이 등록 흐름으로 연결된다.
- **우선순위**: Could
- **관련 이슈**: #37

### FR-MEME-06 공유하기
- **설명**: 폴라로이드 카드를 이미지로 내보내거나 공유 링크를 생성할 수 있다.
- **우선순위**: Could

## 9. 품종 탐색

### FR-BREEDS-01 품종 목록 조회
- **설명**: 67종 품종을 카드 그리드로 표시한다. `GET /api/v1/breeds` 호출. 미인증 접근 허용.
- **우선순위**: Should

### FR-BREEDS-02 품종 검색
- **설명**: 검색창에 품종명을 입력하면 `GET /api/v1/breeds?q=` 호출로 필터링된 목록을 표시한다.
- **우선순위**: Should

## 10. 비기능 요구사항

### NFR-01 스트리밍 응답 지연
- AI 응답 첫 토큰까지 3초 이내

### NFR-02 인증 보안
- NextAuth JWT는 httpOnly 쿠키로 저장. XSS 방지.
- FastAPI는 모든 인증 필요 엔드포인트에서 Bearer 토큰 검증

### NFR-03 PII 마스킹
- 연락처, 주소 등 개인정보는 로그에 마스킹 처리
- **관련 이슈**: #22

### NFR-04 Graceful Fallback
- AI 에이전트 오류 시 사용자에게 "잠시 후 다시 시도해주세요" 메시지 표시
- **관련 이슈**: #24

### NFR-05 반응형 (모바일 대응)
- 최소 지원 해상도: 375px (iPhone SE 기준)
- 채팅 페이지는 모바일에서 단일 컬럼으로 전환
